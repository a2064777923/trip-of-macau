## 1. 修復記錄

### 1.1 路由註冊修復 (2026-04-10)

**問題**：`app.json` 報錯 `["tabBar"][3]["pagePath"]: "pages/tips/index" need in ["pages"]`

**原因**：新增 `pages/tips/detail/index` 頁面時，`pages/tips/index` 被意外從 `pages` 數組中移除

**修復**：在 `app.config.ts` 的 `pages` 數組中補回 `pages/tips/index`

```diff
  pages: [
    'pages/index/index',
    'pages/map/index',
+   'pages/tips/index',
    'pages/tips/detail/index',
    'pages/discover/index',
    ...
  ]
```

### 1.2 狀態層修復 (2026-04-10)

**問題**：`ReferenceError: Cannot read property 'includes' of undefined`

**原因**：舊緩存狀態結構不完整，新代碼對陣列字段調用 `.includes()` 時遇到 `undefined`

**修復**（`gameService.ts`）：
- 新增 `normalizeState(rawState)` 函數，為舊緩存自動補齊默認值
- 補回 `loadGameState()` / `saveState()` 導出函數
- 所有陣列字段（`badges`、`collectedStampIds`、`completedStoryIds`、`completedChapterIds`）均做 `Array.isArray()` 兼容

### 1.3 首頁崩潰修復 (2026-04-10)

**問題**：`ReferenceError: recommendation is not defined`

**原因**：重構首頁評估區時漏掉了 `recommendation` / `featuredReward` 的聲明

**修復**（`index.tsx`）：
```diff
+ const recommendation = useMemo(() => getTravelRecommendation(state.travelAssessment), [state])
+ const featuredReward = rewards[0]
  const selectedInterestCount = assessment.interests.length
```

### 1.4 地圖頁函數結構修復 (2026-04-10)

**問題**：多次 patch 後 `map/index.tsx` 函數區斷裂，`stopArrivalAudio` 和 `playArrivalAudio` 聲明丟失

**修復**：補回完整的函數聲明區塊

### 1.5 外部資源清理 (2026-04-10)

**移除的內容**：
- 地圖頁外部音頻 URL（`cdn.jsdelivr.net`），改為本地 toast 提示
- 地圖頁 marker 外部 iconPath 地址
- 解決了「合法域名不在列表中」和 timeout 問題

### 1.6 微信登入可用性修復 (2026-04-10)

**問題**：此前可用的微信登入在本輪調整後變得不穩，個人頁點擊登入後常只出現失敗提示。

**根因分析**：
- 目前驗收鏈路為了讓微信開發者工具 / Minium MCP 可打開 `dist` 產物，臨時補了一份 `project.config.json`
- 該文件中的 `appid` 目前為占位值 `touristappid`，不是原先真實可用的小程序 `appid`
- 在開發者工具中，這會直接影響 `getUserProfile` / `login` 的真實微信身份授權表現
- 原登入函數缺少「開發態回退策略」，一旦授權鏈路不完整，就只會提示失敗

**修復策略**（`gameService.ts`）：
- 保留 **真實微信授權優先**：仍先調用 `Taro.login()` + `Taro.getUserProfile()`
- 加入 **開發者工具降級登入**：若當前平台為 `devtools` 且授權資料未完整返回，仍會生成可用的本地旅人身份，保證按鈕可用、狀態可回寫
- 個人頁失敗提示改為更明確的 `授權未完成，請再試一次`

**後續建議**：
- 若要完全恢復「和之前一樣」的真實微信登入能力，需要把原先可用的小程序 `appid` 補回 `project.config.json`
- 目前版本已恢復 **可用性**，但在占位 `appid` 下仍屬開發態兼容方案，不等同於正式微信登入上線形態
- 2026-04-10 之後已改為使用真實 `appid`：`wx36e47b792a068165`
- 已新增 `scripts/write-weapp-project-config.js`，並把 `build:weapp` 改成構建後自動寫回 `dist/project.config.json`，避免每次構建後工程配置丟失
- Minium MCP 目前仍卡在 **launch timeout**，但已確認不再是 `project.config.json` 缺失；剩餘問題集中在微信開發者工具自動啟動 / attach 階段

      }
    }
  }
