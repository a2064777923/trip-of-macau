# 澳小遊后台管理系统 — 数据库设计 v2.0

> **版本**: v2.0  
> **日期**: 2026-04-08  
> **状态**: 正式版底座（可扩展、可演进）  
> **数据库引擎**: InnoDB (CloudBase MySQL 8.0)  
> **字符集**: utf8mb4 / utf8mb4_unicode_ci  

---

## 一、设计原则

1. **以数据库为核心**：所有业务能力先在数据模型中落地，再推 API 和 UI。
2. **多城市优先**：每张与地理位置相关的表都带 `city_code` 维度。
3. **层级化 POI**：POI 支持父子关系，支持室内/室外区分。
4. **空间扩展**：预留经纬度索引、地图坐标、瓦片坐标、室内坐标体系。
5. **状态机统一**：所有核心实体都有 `status` + `created_at`/`updated_at` + `_openid`。
6. **审计就绪**：关键操作可被 `audit_logs` 追溯。
7. **RBAC 内建**：管理员/角色/权限/菜单权限/API 权限/数据范围。

## 十一、命名约定

| 约定 | 示例 |
|------|------|
| 表名 | 小写下划线复数: `story_chapters` |
| 主键 | `id BIGINT AUTO_INCREMENT` |
| 外键 | `{table}_id` |
| 状态字段 | `status TINYINT` (0=禁用/草稿, 1=启用, 2=归档) |
| 时间戳 | `created_at`, `updated_at`, `deleted_at`(软删), `published_at`(发布) |
| 枚举 | MySQL ENUM 或 VARCHAR + 注释 |
| JSON 字段 | 用于数组/嵌套/动态配置 |
| CloudBase 要求 | 每张表必须有 `_openid VARCHAR(64) DEFAULT '' NOT NULL` |
