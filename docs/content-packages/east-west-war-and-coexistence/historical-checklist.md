# 東西方文明的戰火與共生：史實與演繹校對清單

## 史實依據

- `route-sites`: 本故事線採用澳門半島可實地到訪的地點作為路線骨架，包括媽閣廟、亞婆井前地、崗頂一帶、大炮台與議事亭前地。
- `ama-temple`: 媽閣廟是澳門早期海岸信仰與航海活動的重要地標，可作為開場的海岸記憶入口。
- `early-portuguese-presence`: 16 世紀中葉葡萄牙人逐步進入澳門並形成居留與貿易活動，相關敘述應寫作「前後」「據史料脈絡」而非單點必然事件。
- `coastal-community`: 澳門早期海岸社群與漁民活動可作為海防、航海、廟宇信仰的背景，但具體人物與道具多屬文學化重構。
- `trade-tax`: 中葡早期通商、居留與納稅安排可作為契約殘頁的史實靈感，但不可寫成已確認存在的單一遊戲道具。
- `lilau-settlement`: 亞婆井前地與葡人早期聚居記憶相關，適合作為「邊界、生活水源、社群形成」的章節錨點。
- `boundary-systems`: 明清官府與葡人在澳門形成長期管治、邊界、居留與貿易協商關係；具體「界碑」與「防線動畫」應標註為敘事化呈現。
- `fortification`: 澳門歷史上有炮台、城防與海防設施，南灣與半島高地的軍事空間可作為防線想像的資料基礎。
- `garrison-life`: 駐守、水源、夜間巡邏等日常軍事生活可作為互動內容的通用歷史情境，具體水壺殘件為文學化道具。
- `church-and-hill`: 崗頂一帶與教堂、劇院、修院及高地視野相關，適合設計「宗教、教育、城市高地視角」與「瞭望」的故事張力。
- `vantage-points`: 高地視角可支持海面觀察與城市空間閱讀，但「特定隱藏瞭望哨」需作為文學演繹。
- `dutch-attack-1622`: 1622 年荷蘭艦隊攻澳與大炮台防守是澳門軍事史的重要節點，可作為第四章核心史實。
- `coexistence`: 澳門長期形成中西文化交流、宗教、建築、商貿與城市治理交疊的歷史景觀，可作為終章主題。
- `artifact-classes`: 令牌、契約殘頁、地圖、拓片、炮彈殘片等是常見博物館敘事符號，可用於收集物設計，但需明確不是宣稱現實存在的同一件文物。
- `none`: 該素材主要服務互動或獎勵演出，沒有直接史實宣稱。

## 文學演繹

- `copper-mirror-motif`: 「殘缺的海防銅鏡」是故事線統一線索，用來串聯五章，不宣稱為真實文物。
- `route-drama`: 章節間路線、支線插入與灰色路徑高亮是小程序故事模式設計，不是歷史事件本身。
- `ch01-reconstruction`: 「鏡海初戰」以早期中葡接觸、海岸防禦、通商協商為史實背景進行戲劇化重構。
- `ch02-boundary-drama`: 「南灣防線」把居留、邊界與水源記憶編排為可互動的邊界拼圖。
- `ch03-watch-drama`: 「山城戒備」把高地視野、宗教建築與城市防務想像編排為瞭望任務。
- `ch04-battle-drama`: 「炮台硝煙」基於 1622 年荷澳戰役設計炮擊動畫、佈防遊戲與守軍道具。
- `finale-witness`: 終章的個人探索報告、數字紀念藏品與「時空見證者」身份是產品敘事層。
- `pickup-fiction`: 拾取物名稱與出現條件為遊戲化設計，不能寫成現實中已考古確認的物件。
- `pickup-glow`: 拾取物發光、粒子、Lottie 閃爍是視覺提示，不承載史實內容。
- `reward-theatre`: 勳章、稱號、全屏獎勵動畫與音效屬遊戲內獎勵演出。

## 需要避免寫死為史實的表述

- 不要寫成「1553 年某日媽閣廟爆發有確切記錄的某場戰役」，應改為「以 16 世紀中葉中葡接觸與海岸防務為背景的文學化重構」。
- 不要寫成「亞婆井前地現存某明葡界碑拓片」，應改為「以邊界協商和聚居記憶為靈感的互動拓片」。
- 不要寫成「崗頂劇院門口曾存在官方秘密軍事瞭望哨」，應改為「利用高地視野設計的城市防務想像」。
- 不要寫成「中西守軍聯名誓詞原件已被找到」，應改為「以荷澳戰役守城記憶為基礎設計的象徵性誓詞」。
- 不要寫死探索度增加百分比；所有探索進度由 Phase 32 的元素權重和已發布元素總量動態計算。
