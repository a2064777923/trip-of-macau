<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> package API | package metadata and provenance are displayed to authenticated operators |
| smoke script -> local services | script uses admin bearer and reads local DB/API state |
| package page -> cross-domain links | UI routes users into existing editors by route only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T33-09 | Information Disclosure | package page | mitigate | Do not display secrets; show only asset ids, object keys, canonical URLs, prompts/scripts intended for operators, and provenance notes. |
| T33-10 | Spoofing | smoke auth | mitigate | Read admin bearer from environment or ignored auth file; do not commit tokens. |
| T33-11 | Tampering | UI delete/edit actions | avoid | Phase 33 page is inspection/navigation-first; destructive editing remains in backend API and existing workbenches, not quick one-click UI deletes. |
</threat_model>
