<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generated/local assets -> admin upload API | local files may be missing, malformed, or oversized |
| manifest docs -> SQL seed | prompt/script text and COS keys become database content |
| helper script -> local credentials | ignored local auth may be read for upload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T33-03 | Information Disclosure | manifest and upload helper | mitigate | Do not store or print API keys/COS credentials; store only object keys, canonical URLs, prompt text, and scripts. |
| T33-04 | Denial of Service | upload helper | mitigate | Skip missing files and rely on backend upload size/type enforcement; do not recursively upload arbitrary directories. |
| T33-05 | Tampering | SQL manifest seed | mitigate | Use deterministic ids and `ON DUPLICATE KEY UPDATE`; import through utf8mb4; keep prompts/scripts in committed docs for review. |
</threat_model>
