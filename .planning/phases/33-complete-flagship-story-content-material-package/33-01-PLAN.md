<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> material package API | operator-supplied metadata, paths, prompts, and JSON enter admin backend |
| material package API -> content assets/story rows | package rows reference existing cross-domain objects by ids and codes |
| manifest/provenance -> repository/COS | local paths and COS keys must not reveal secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T33-01 | Tampering | `AdminStoryMaterialPackageServiceImpl` | mitigate | Filter every read/write by `deleted=0`, validate package ownership for item updates/deletes, recompute counters server-side instead of trusting client counters. |
| T33-02 | Information Disclosure | package manifest fields | mitigate | Store COS object keys and canonical URLs only; do not store API keys, COS secret ids, provider keys, or local absolute paths containing secrets. |
| T33-03 | Repudiation | package create/update | mitigate | Persist `created_by_admin_id` and `created_by_admin_name` on create and keep timestamps from BaseEntity conventions. |
</threat_model>
