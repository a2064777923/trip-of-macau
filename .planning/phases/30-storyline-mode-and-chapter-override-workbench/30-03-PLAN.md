<objective>
Build the dedicated admin Storyline Mode Workbench UI for route arrangement and chapter-level inherit/disable/replace/append override authoring, using structured cards and previews instead of raw JSON-first forms.
</objective>

<threat_model>
| Risk ID | ASVS L1 risk | Surface | Mitigation |
| --- | --- | --- | --- |
| T30-03-01 | Operator misconfiguration of required story route/anchor fields | Workbench forms | Use required field rules, validation findings panel, and `focusFirstInvalidField` before submit. |
| T30-03-02 | UI state corruption from nested override editing | Chapter tree and override editor | Keep selected chapter/selected override state isolated; reload snapshot after each save/delete. |
| T30-03-03 | Unauthorized route exposure | `App.tsx` and sidebar | Register under existing protected admin shell only. |
| T30-03-04 | Raw JSON misuse | Advanced panels | Keep `進階 JSON` collapsed and disabled by default; primary cards must generate structured payloads. |
| T30-03-05 | Misleading public preview | Runtime preview panel | Display `publicRuntimePath` and compiled preview from backend, not local guessed output. |
</threat_model>
