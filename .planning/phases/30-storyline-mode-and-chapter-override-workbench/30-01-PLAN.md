<objective>
Add a dedicated admin backend facade for storyline mode and chapter override authoring so the admin UI can load a complete workbench snapshot, update story-mode config, assign chapter anchors, and author per-step inherit/disable/replace/append overrides without raw JSON as the primary contract.
</objective>

<threat_model>
| Risk ID | ASVS L1 risk | Surface | Mitigation |
| --- | --- | --- | --- |
| T30-01-01 | Broken access control on story-mode writes | `/api/admin/v1/storylines/{storylineId}/mode-workbench/**` | Keep all endpoints under existing admin auth path `/api/admin/v1/**`; do not add public write routes. |
| T30-01-02 | Runtime contract drift from parallel story storage | Storyline mode facade | Persist through `story_chapters`, `experience_flows`, `experience_flow_steps`, `experience_overrides`, and `content_relation_links`; do not create a story-only override table. |
| T30-01-03 | Malformed JSON or policy injection | Story-mode config and override payloads | Require JSON object payloads with `schemaVersion: 1` for `storyModeConfigJson`, `overridePolicyJson`, and override configs. |
| T30-01-04 | Invalid anchor references | Chapter anchor update endpoint | Validate each anchor type and target before save; return deterministic 4xx for missing POI/indoor/task/manual target rules. |
| T30-01-05 | Public runtime leakage of drafts | Override and replacement step persistence | Default new objects to `draft`; publish only when explicit `status='published'`; smoke tests must verify public runtime uses published compiled data. |
</threat_model>

<action>
Create the admin storyline mode facade with these exact classes and routes:

- `AdminStorylineModeController` under `@RequestMapping("/api/admin/v1/storylines/{storylineId}/mode-workbench")`.
- `GET ""` returns `AdminStorylineModeResponse.Snapshot`.
- `PUT "/mode-config"` accepts `AdminStorylineModeRequest.StoryModeConfigUpsert` and returns `Snapshot`.
- `PUT "/chapters/{chapterId}/anchor"` accepts `AdminStorylineModeRequest.ChapterAnchorUpsert` and returns `Snapshot`.
- `PUT "/chapters/{chapterId}/overrides"` accepts `AdminStorylineModeRequest.ChapterOverridePolicyUpsert` and returns `Snapshot`.
- `POST "/chapters/{chapterId}/override-steps"` accepts `AdminStorylineModeRequest.OverrideStepUpsert` and returns `AdminStorylineModeResponse.OverrideRule`.
- `PUT "/chapters/{chapterId}/override-steps/{overrideId}"` accepts `AdminStorylineModeRequest.OverrideStepUpsert` and returns `AdminStorylineModeResponse.OverrideRule`.
- `DELETE "/chapters/{chapterId}/override-steps/{overrideId}"` soft-deletes the override and returns `Boolean.TRUE`.
- `GET "/runtime-preview"` returns `AdminStorylineModeResponse.RuntimePreview`.

Create `AdminStorylineModeService` and `AdminStorylineModeServiceImpl`. The service must assemble a snapshot with:

- `storyline`: existing `AdminStoryLineDetailResponse`.
- `chapters`: ordered chapter summaries.
- `routeStrategy`: structured story-mode route and branch policy parsed from `storyModeConfigJson`.
- `chapterRuntimes`: one item per chapter containing `chapter`, `anchor`, `inheritedFlow`, `chapterFlow`, `overrides`, `compiledStepPreview`, and `validationFindings`.
- `availableAnchorTypes`: exactly `poi`, `indoor_building`, `indoor_floor`, `indoor_node`, `task`, `overlay`, `manual`.
- `availableOverrideModes`: exactly `inherit`, `disable`, `replace`, `append`.
- `publicRuntimePath`: `/api/v1/storylines/{storylineId}/runtime`.

Implement structured request fields:

- `StoryModeConfigUpsert`: `hideUnrelatedContent`, `nearbyRevealEnabled`, `nearbyRevealRadiusMeters`, `currentRouteHighlight`, `inactiveRouteStyle`, `clearTemporaryProgressOnExit`, `preservePermanentEvents`, `branchSourceType`, `branchInsertPosition`, `branchSkippable`, `branchAffectsStoryProgress`, `manualBranchPoiIds`, `advancedJsonEnabled`, `advancedStoryModeConfigJson`.
- `ChapterAnchorUpsert`: `anchorType`, `anchorTargetId`, `anchorTargetCode`, `anchorLabelOverride`, `routeOrder`, `routeSegmentStyle`.
- `ChapterOverridePolicyUpsert`: `inheritDefaultFlow`, `disableDefaultArrivalMedia`, `appendStorySpecificRewards`, `advancedJsonEnabled`, `advancedOverridePolicyJson`.
- `OverrideStepUpsert`: `targetStepCode`, `overrideMode`, `replacementStepId`, `replacementStepDraft`, `effectPreset`, `mediaAssetId`, `rewardRuleIds`, `pickupCodes`, `challengeCode`, `explorationWeightLevel`, `sortOrder`, `status`, `advancedJsonEnabled`, `advancedOverrideConfigJson`.

The service must:

- Require the storyline and chapter exist and match.
- Use `schemaVersion: 1` in every generated `storyModeConfigJson`, `overridePolicyJson`, and `overrideConfigJson`.
- Normalize story-mode config to include both Phase 30 canonical keys and public-runtime compatibility keys:
  - `nearbyRevealRadiusMeters` and `nearbyRevealMeters`.
  - `currentRouteHighlight` and `currentRouteStyle`.
  - `clearTemporaryProgressOnExit` and `exitResetsSessionProgress`.
- Validate override mode values against `inherit`, `disable`, `replace`, `append`.
- Persist override rows as `ownerType='story_chapter'`, `ownerId=chapterId`, `targetOwnerType=anchorType`, `targetOwnerId=anchorTargetId`, `targetStepCode`, `overrideMode`, `replacementStepId`, `overrideConfigJson`, `status`, and `sortOrder`.
- For `replacementStepDraft`, create or update a chapter-specific `experience_flow` with `flowType='story_chapter'`, `mode='storyline'`, `owner` linked through the chapter `experienceFlowId`, then create a replacement `experience_flow_step` and use its ID in the override.
- Keep advanced JSON collapsed/fallback only: if `advancedJsonEnabled=false`, ignore advanced JSON fields; if true, validate and require `schemaVersion`.
</action>
