<objective>
Align the public storyline runtime DTO with Phase 30 story-mode and chapter override authoring so the mini-program-facing API can return route strategy, chapter anchor metadata, inherited flows, override rules, and compiled steps using stable runtime keys.
</objective>

<threat_model>
| Risk ID | ASVS L1 risk | Surface | Mitigation |
| --- | --- | --- | --- |
| T30-02-01 | Public data leakage | Runtime DTO serialization | Keep DTO fields runtime-focused; do not add admin-only `status`, draft editor state, or internal validation details to public response. |
| T30-02-02 | Client breakage from key drift | Story-mode config parsing | Support both Phase 30 canonical keys and existing Phase 28 keys; document canonical public output names. |
| T30-02-03 | Unauthorized state mutation | Public storyline runtime endpoint | Keep `GET /api/v1/storylines/{id}/runtime` read-only; event/session mutation remains behind existing authenticated endpoints. |
| T30-02-04 | Broken route because draft rows are compiled | Flow/override selection | Continue filtering public flows, steps, templates, and overrides to `status='published'`. |
| T30-02-05 | Denial from malformed config JSON | Runtime JSON parsing | Treat malformed optional JSON as empty maps and never throw public 500 for optional editor config. |
</threat_model>

<action>
Harden storyline compiled step output for Phase 30 override semantics.

Ensure `PublicExperienceServiceImpl.compileSteps` and `applyOverride` support:

- `disable`: remove the inherited step whose key matches `targetStepCode`.
- `replace`: replace the inherited step whose key matches `targetStepCode` with `replacementStepId`.
- `append`: append the replacement step immediately after the target step when it exists, otherwise append to the end.
- `inherit`: no mutation; retain inherited step.

Ensure step matching uses `stepCode` first and falls back to `step-{id}` only when `stepCode` is blank.

Ensure public output never serializes `status` for flows, steps, templates, or overrides. If DTOs already omit status, add smoke/source checks rather than adding code.
