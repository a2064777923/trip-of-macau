<objective>
Build the dedicated POI default experience workbench UI so operators can configure natural walk-in and tap-to-explore flows through structured cards, timeline editing, media/reward selectors, and reusable templates rather than generic flow CRUD or raw JSON-first forms.
</objective>

<threat_model>
| Risk ID | ASVS L1 risk | Surface | Mitigation |
| --- | --- | --- | --- |
| T29-02-01 | Operator misconfiguration through hidden required fields | Workbench forms | Use required card sections, bottom validation strip, and `focusFirstInvalidField` scroll/focus/shake before save. |
| T29-02-02 | UI state corruption from large nested forms | Timeline and property editor | Keep selected step state isolated, debounce derived preview only if needed, and reload snapshot after save/delete/template actions. |
| T29-02-03 | Unauthorized or public route exposure | `App.tsx` and `DefaultLayout.tsx` | Register routes only under the existing protected admin layout. |
| T29-02-04 | Raw JSON injection into visual preview | Structured card preview | Render structured values as escaped text/selectors/cards; keep advanced JSON inside controlled text areas only. |
| T29-02-05 | Cross-domain confusion | POI page and generic experience page | Label the page as "POI 地點體驗工作台" and keep generic governance/template routes separate. |
</threat_model>

<action>
Create `packages/admin/aoxiaoyou-admin-ui/src/pages/POIExperienceWorkbench/index.tsx` and `index.scss`.

The page must:

- Read `poiId` from route param `/space/pois/:poiId/experience`.
- Also support `/space/poi-experience` by showing a POI selector before loading a snapshot.
- Load snapshot through `getAdminPoiExperienceDefault`.
- Render a three-panel workbench:
  - Left: POI selector, flow summary, timeline list of ordered steps, quick-add buttons.
  - Middle: visual step composition grouped under `點擊與前往`, `抵達與媒體`, `打卡與任務`, `拾取與隱藏`, `完成與獎勵`.
  - Right: selected-step property editor with structured sections.
  - Bottom: validation findings, public runtime path, missing media/reward hints, and unpublished-step warnings.
- Provide canonical quick-add presets for:
  - `tap_intro` as `intro_modal`.
  - `start_route_guidance` as `route_guidance`.
  - `arrival_intro_media` as `proximity_media`.
  - `release_checkin_tasks` as `checkin_task`.
  - `pickup_side_clues` as `pickup`.
  - `hidden_dwell_achievement` as `hidden_challenge`.
  - `completion_reward_title` as `reward_grant`.
- Use structured cards for:
  - 條件卡: trigger preset, radius meters, dwell seconds, after-step, once-per-user, time window, required item/badge codes.
  - 效果卡: modal title/body, primary action, route card types, task codes, pickup codes, reward summary.
  - 媒體卡: media asset, full-screen media asset, audio asset, asset preview.
  - 獎勵卡: reward rule IDs, reward summary, exploration weight level, required-for-completion.
- Keep a collapsed `進階 JSON` section with `advancedJsonEnabled` and text areas for trigger/condition/effect JSON.
- Save through Plan 01 API helpers and reload snapshot after success.
- Delete steps with confirmation.
- Save selected step as template through `saveAdminPoiExperienceStepAsTemplate`.
- Use Traditional Chinese copy only.
- Call `focusFirstInvalidField(form)` or the existing project helper on validation failure, and ensure an invalid card visibly shakes or receives an error class.

Do not add raw JSON as the first visible editor. Do not put Phase 30 chapter override controls in this page.
</action>
