<objective>
Add a dedicated admin POI experience facade and matching frontend contract so the POI workbench can load, upsert, and template-save a POI's default walk-in flow atomically while still using the Phase 28 canonical experience tables.
</objective>

<threat_model>
| Risk ID | ASVS L1 risk | Surface | Mitigation |
| --- | --- | --- | --- |
| T29-01-01 | Broken access control on POI experience writes | `/api/admin/v1/pois/{poiId}/experience/**` | Keep endpoints under `/api/admin/v1/**` and do not add public or unauthenticated bypass routes. |
| T29-01-02 | Contract drift from parallel POI storage | Admin facade service | Persist only through `experience_flows`, `experience_flow_steps`, `experience_bindings`, `experience_templates`, and `exploration_elements`; reject non-POI binding shapes in this facade. |
| T29-01-03 | Input tampering through arbitrary JSON | Structured step upsert and advanced JSON fallback | Compile structured fields into `schemaVersion: 1` payloads and reuse/replicate Phase 28 versioned-object validation before persistence. |
| T29-01-04 | Public runtime mismatch | POI binding creation/update | Enforce one published default binding per POI/flow role and verify the binding can be consumed by existing public `GET /api/v1/experience/poi/{poiId}`. |
| T29-01-05 | Template poisoning or misleading reuse | Save-step-as-template endpoint | Template type must be derived from canonical step semantics and persisted with canonical `templateType`, `riskLevel`, and `status`. |
</threat_model>

<action>
Create a POI experience facade with these exact classes and routes:

- `AdminPoiExperienceController` under `@RequestMapping("/api/admin/v1/pois/{poiId}/experience")`.
- `GET /default` returns `AdminPoiExperienceResponse.Snapshot`.
- `PUT /default-flow` accepts `AdminPoiExperienceRequest.FlowUpsert` and returns `Snapshot`.
- `POST /steps` accepts `AdminPoiExperienceRequest.StepStructuredUpsert` and returns `AdminPoiExperienceResponse.Step`.
- `PUT /steps/{stepId}` accepts `AdminPoiExperienceRequest.StepStructuredUpsert` and returns `AdminPoiExperienceResponse.Step`.
- `DELETE /steps/{stepId}` soft-deletes the step and returns `Boolean.TRUE`.
- `POST /steps/{stepId}/save-template` accepts `AdminPoiExperienceRequest.SaveTemplateRequest` and returns `AdminExperienceResponse.Template`.

Create `AdminPoiExperienceService` and `AdminPoiExperienceServiceImpl`. The service must:

- Require the POI exists before any operation.
- Load or create a single default flow with `flowType="default_poi"`, `mode="walk_in"`, `status` defaulting to `"draft"`, and `code` defaulting to `"poi_" + poiId + "_default_walk_in"` when the operator does not provide one.
- Ensure a binding row with `ownerType="poi"`, `ownerId=poiId`, `bindingRole="default_experience_flow"`, `inheritPolicy="inherit"`, `priority=0`, and the selected flow ID.
- Preserve existing Phase 28 columns and soft-delete behavior; do not create new tables.
- Sort steps by `sortOrder ASC, id ASC`.
- Return snapshot fields: `poi`, `flow`, `binding`, `steps`, `templates`, `validationFindings`, and `publicRuntimePath`.
- Compile structured step fields into `triggerConfigJson`, `conditionConfigJson`, `effectConfigJson`, and `rewardRuleIdsJson`.
- Always include `schemaVersion: 1` in `triggerConfigJson`, `conditionConfigJson`, and `effectConfigJson`.
- Keep `rewardRuleIdsJson` as a JSON array or object according to Phase 28 behavior; do not force it to be versioned if the existing contract treats it as non-versioned.
- Allow advanced JSON only when `advancedJsonEnabled=true`, and still reject malformed JSON or versioned fields without `schemaVersion`.

The structured step request must include concrete fields for:

- Base fields: `stepCode`, `stepType`, `templateId`, `stepNameZh`, `stepNameZht`, `descriptionZh`, `descriptionZht`, `triggerType`, `mediaAssetId`, `explorationWeightLevel`, `requiredForCompletion`, `status`, `sortOrder`.
- Trigger fields: `triggerPreset`, `triggerRadiusMeters`, `dwellSeconds`, `tapActionCode`, `afterStepCode`.
- Condition fields: `conditionPreset`, `oncePerUser`, `timeWindowStart`, `timeWindowEnd`, `requiredItemCodes`, `requiredBadgeCodes`.
- Effect fields: `effectPreset`, `modalTitle`, `modalBody`, `primaryActionLabel`, `routeCardTypes`, `taskCodes`, `pickupCodes`, `rewardRuleIds`, `rewardSummary`, `fullScreenMediaAssetId`, `audioAssetId`.
- Advanced fields: `advancedJsonEnabled`, `advancedTriggerConfigJson`, `advancedConditionConfigJson`, `advancedEffectConfigJson`.

Map these canonical step types:

- `intro_modal`
- `route_guidance`
- `proximity_media`
- `checkin_task`
- `pickup`
- `hidden_challenge`
- `reward_grant`
- `custom`
</action>
