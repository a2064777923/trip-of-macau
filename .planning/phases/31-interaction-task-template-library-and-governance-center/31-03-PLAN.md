<objective>
Replace the current thin template and governance tabs with dedicated Traditional Chinese operator workbenches for reusable interaction/task templates and cross-domain governance. The UI must expose structured presets and filters first, with advanced JSON only as a secondary escape hatch.
</objective>

<threat_model>
## T31-03 - Unsafe Admin Authoring UX

Threat: Operators publish high-risk template payloads by pasting raw JSON without seeing usage or conflict impact.

Mitigation:
- Show risk labels and schema guard hints in the template editor.
- Expose usage count and conflict count before editing.
- Keep advanced JSON collapsed and provide preset buttons to fill safe schema-versioned payloads.

Threat: Governance filters are unclear, causing operators to miss conflicts.

Mitigation:
- Every filter must have a visible Traditional Chinese label.
- `conflictOnly` and `highRiskOnly` toggles must be explicit switches.
</threat_model>
