## T31-04 - Verification and Seed Integrity

Threat: Chinese seed data is corrupted by Windows shell encoding.

Mitigation:
- Write SQL as a UTF-8 file.
- Include `SET NAMES utf8mb4;`.
- Import using `--default-character-set=utf8mb4`.

Threat: Smoke script accidentally uses stale services or fake mocks.

Mitigation:
- Smoke script must call live `127.0.0.1:8081` admin backend endpoints.
- Smoke script must assert seeded template codes, usage refs, and conflict finding types from responses.
</threat_model>
