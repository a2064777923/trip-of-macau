<objective>
Turn the existing experience workbench into the Phase 28 operator foundation so story and content control-plane work stops routing through placeholder surfaces or raw payload editing.

Purpose: Apply D-07 and D-13 through D-16 to the admin UI while keeping LINK-01 contract alignment with the backend files hardened in Plan 01.
Output: Typed admin UI contracts, reachable workbench routes, a three-panel Traditional Chinese foundation UX, and reusable content/media surfaces with assembled preview before publish.
</objective>

<threat_model>
| Risk ID | ASVS L1 risk | Surface | Mitigation |
| --- | --- | --- | --- |
| T28-02-01 | UI-driven tampering through mismatched payload shapes | `api.ts` and `types/admin.ts` | Mirror backend DTO names and required fields exactly so the UI cannot silently post stale or local-only contract shapes. |
| T28-02-02 | Operator error from unclear validation and hidden failures | Workbench forms and drawers | Keep scroll-to-error, focus-first-invalid, and visible validation/governance hints in the workbench instead of silent submit failures. |
| T28-02-03 | Unauthorized navigation exposure | Admin routing and layout | Keep all workbench routes inside the existing protected admin shell; do not add public or bypass routes. |
| T28-02-04 | Content injection via rendered rich payloads | Workbench preview/state surfaces | Render payloads as controlled form input/state, not arbitrary HTML injection; keep JSON editing as escaped text input only. |
| T28-02-05 | Content/media mismatch before publish | Story content blocks, media library, chapter workbench preview | Reuse the existing content-block and media surfaces, then surface an assembled pre-publish preview with missing-asset or draft-state warnings before chapter publication. |
</threat_model>
