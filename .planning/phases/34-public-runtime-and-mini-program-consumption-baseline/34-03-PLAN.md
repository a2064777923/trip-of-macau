## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime DTO -> story page | traveler-visible content and interaction metadata |
| story page -> event API | passive read events become stateful only when authenticated |
| media URL -> renderer | remote media may fail or be unavailable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T34-07 | Denial of Service | story page runtime fetch | mitigate | Keep fallback story data visible and show `使用本機快取` instead of blocking the page. |
| T34-08 | Spoofing | passive event reporting | mitigate | Anonymous read events no-op; authenticated writes use existing bearer token only. |
| T34-09 | Denial of Service | media rendering | mitigate | Missing/failing media shows placeholders or Lottie fallbacks, not blank screens. |
</threat_model>
