<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mini-program -> public runtime API | story data is read anonymously or with token when available |
| mini-program -> event/session APIs | stateful writes require bearer token |
| local fallback -> live runtime | cached data is fallback only, not the preferred runtime source |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T34-04 | Spoofing | story events | mitigate | API helpers use existing token header; anonymous event reporting no-ops instead of fabricating identity. |
| T34-05 | Tampering | repeated events | mitigate | `recordStoryRuntimeEvent` generates stable `clientEventId` values so backend idempotency works. |
| T34-06 | Information Disclosure | client fallback | mitigate | Fallback uses already public story list data, not admin material package/private data. |
</threat_model>
