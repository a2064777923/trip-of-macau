<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public backend -> anonymous mini-program | Read-only story runtime is public traveler content |
| admin-authored JSON -> public DTO | Backend compiles and labels validated payloads before client consumption |
| runtime DTO -> mini-program renderer | Client renders only traveler-safe metadata and media URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T34-01 | Information Disclosure | public runtime DTO | mitigate | Add traveler-safe metadata fields without exposing secrets, admin audit logs, provider config, or unreviewed draft content. |
| T34-02 | Tampering | lifecycle/status filtering | mitigate | Enforce `published` filters for public storylines, chapters, flows, steps, overrides, and content where status exists. |
| T34-03 | Denial of Service | runtime DTO size | mitigate | Keep runtime fields bounded to current story/chapters/steps; do not attach full admin package manifests to runtime responses. |
</threat_model>
